Introduction

CopiX semble correctement installé sur votre poste.

Si vous voyez des messages d'erreurs, assurez vous d'avoir créé tous les répertoires temporaires requis, et que PHP ait les droits nécessaires sur ces répertoires.

Pour plus de détails sur l'installation de copix, consultez la documentation, section installation.

Utilisation

CopiX est distribué avec un module exemple simple, vous permettant de découvrir comment développer une première page.

Si vous souhaitez d'autres exemples, ainsi que des tutoriels, vous pouvez consulter la documentation officielle de CopiX.

  • Apprendre les principes de bases de Copix accompagné de son Tutoriel "hello You"
  • Savoir ce qu'est la coordination : les descriptifs d'actions, les actiongroups, les actions.
  • Utiliser des zones, des templates.
  • Accéder à une base de donnée avec CopixDB et CopixDAO.
  • Suivre le tutoriel "News" récapitulant tous ces concepts.

Actions systématiques & Cadre principal

Si vous souhaitez modifier l'aspect du cadre principal de l'application, vous pouvez le faire en modifiant le template principal, et en modifiant le coordinateur de l'application.

Apprenez donc à modifier le processus standard de Copix.

Développement de modules

Les modules sont des "briques" applicatives, destinées à être intégrés très facilement dans des projets plus importants. Ces modules profitent du socle CopiX pour interragir avec l'application.

Consulter la documentation pour en savoir plus sur les modules

Allez plus loin

Une fois les connaissances de bases acquises, vous pourrez utiliser les fonctionnalités avancées de Copix, pour faire de veritables applications et site web, comme :

  • développer des plugins, ces petites briques techniques dédiées à des actions systématiques, ajoutant des fonctionnalités à Copix.
  • Internationnaliser son site Copix
  • Rendre son application paramétrable
  • Apporter une gestion d'utilisateurs et de droits
  • Profiter du système évènementiel
  • Et plein d'autres choses...
